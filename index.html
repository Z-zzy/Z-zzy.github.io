小白的探索之路

在leetcode上先从数组开始做起，下面的这个题是比较容易的一个题。但是这个思路确实很棒，值得记录下~

题目是这样滴~

448. Find All Numbers Disappeared in an Array

Given an array of integers where 1 ≤ a[i] ≤ n (n = size of array), some elements appear twice and others appear once.

Find all the elements of [1, n] inclusive that do not appear in this array.

Could you do it without extra space and in O(n) runtime? You may assume the returned list does not count as extra space.

我自己原先的想法

其实一开始看到这个题的时候感觉没什么难度的，但是看到最后，发现要求是O（n）的时间复杂度和没有额外空间的时候真的有点懵。

毕竟我的第一想法就是for循环，但是显然不行。如果是这样的话，就得从题上给的这个数组做文章了。

注意到了题目中的这个条件，所有的元素都在1到n的区间内，而且这个向量里一共就n个元素。这好像就是在暗示用内容当作下标访问元素啊。

接下来就是做标记了，标记一下内容当下标指向的元素，最后再统计一下所做的标记就可以出结果了。

写在最后

其实用内容当作下标的话是有一定隐患的，如果向量中的数据被污染或者传入前没有检验的话会在运行中出现错误。
所以，在这样使用的时候一定要做好相应的检测以及防范，可以在运行算法之前先过一个循环判断下是否会有越界的可能。
